# 浏览器下载更新说明

当前版本的更新策略为“远程版本清单检查 + 浏览器下载”。

## 远程版本清单

客户端默认会按顺序尝试读取以下地址上的 `update-manifest.json`：

1. `TN_AUDIO_UPDATE_CONFIG_URLS` 或 `TN_AUDIO_UPDATE_CONFIG_URL` 环境变量中配置的地址
2. `https://gitee.com/lingyu_mayun/TN_Audio_Check_Tools/raw/master/TN_Audio_Tools/update-manifest.json`
3. `https://cdn.jsdelivr.net/gh/JohnsonJinyu/TN_Audio_Check_Tools@master/TN_Audio_Tools/update-manifest.json`
4. `https://ghfast.top/https://raw.githubusercontent.com/JohnsonJinyu/TN_Audio_Check_Tools/master/TN_Audio_Tools/update-manifest.json`
5. `https://raw.githubusercontent.com/JohnsonJinyu/TN_Audio_Check_Tools/master/TN_Audio_Tools/update-manifest.json`

建议后续将该 JSON 托管到国内可达的静态地址，并通过环境变量覆盖默认地址。

## JSON 字段

`version`
最新版本号，支持 `1.2.3` 或 `v1.2.3`。

`releaseName`
展示在设置页中的版本标题。

`releaseDate`
ISO 日期字符串，用于展示发布时间。

`releaseNotes`
字符串或字符串数组，会展示在设置页“版本说明”中。

`downloadUrl`
浏览器打开的下载页或发布页地址。

`mirrorUrl`
可选。存在时，浏览器下载会优先打开该地址。

`mandatory`
保留字段，当前仅做展示预留。

## 发布步骤

1. 打包新的安装版。
2. 上传安装包到 GitHub Release、公司内网或其他浏览器可访问地址。
3. 更新 `update-manifest.json` 中的版本号、说明和下载地址。
4. 将更新后的 JSON 发布到客户端可访问的静态地址。
5. 在安装版应用中点击“检查更新”验证结果。
