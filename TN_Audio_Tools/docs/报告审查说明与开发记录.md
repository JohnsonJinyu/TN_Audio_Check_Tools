## 第一部分：功能说明

### 功能概述

报告审查功能用于对 Acqua 输出的 Word 报告进行快速结构化检查，帮助工程师在人工复核前先发现明显问题。

当前版本主要提供以下能力：

- 选择本地 `.doc` 或 `.docx` 报告并发起审查
- 自动执行 8 项基础检查
- 将审查结果保存到页面历史记录，而不是只停留在一次性弹窗里
- 支持从历史记录再次打开详情，并清空历史列表

### 使用方式

#### 1. 进入页面

在应用左侧导航点击 **报告审查**，进入审查页面。

#### 2. 选择本地报告

页面顶部提供一个居中的文件选择区：

```text
┌────────────────────────────────────┐
│        点击或拖拽选择文件          │
│       支持 .doc 和 .docx 格式      │
└────────────────────────────────────┘
```

支持两种方式：

- 点击选择区，打开本地文件选择对话框
- 直接将 `.doc` / `.docx` 文件拖入选择区

选择成功后，页面会显示当前已选文件名。

#### 3. 开始审查

点击右侧 **开始审查** 按钮后，系统会执行以下流程：

1. 校验文件路径与文件格式
2. 若为 `.doc`，先在后台转换为临时 `.docx`
3. 解析 Word 内容并构造统一审查数据
4. 执行 8 项检查逻辑
5. 生成结构化审查结果并写入历史记录

#### 4. 查看结果与历史

当前版本的结果查看方式如下：

- 页面下方会保留最近审查记录
- 每条历史都可以点击 **查看详情** 打开详细结果
- 支持 **清空列表** 删除本地历史记录

说明：

- 结果不会因为关闭详情窗口而丢失
- 历史记录保存在浏览器本地存储中，仅在当前机器当前环境下可见

### 支持的文件格式

- `.doc`
- `.docx`

### 当前审查项

系统当前固定执行以下 8 项检查：

1. 目录提取
2. 目录页数检查
3. 章节与目录对应
4. 报告基本信息一致性
5. 测试项一致性
6. 名称污染检查
7. 测试人员信息识别
8. POLQA 配置检查

### 审查结果状态说明

每项检查和整体结果都使用以下状态：

- `pass`：通过
- `warning`：有警告
- `review`：需人工复核
- `error`：有错误

### 当前限制

1. `.doc` 报告仍依赖后台转换链路，极端格式下可能只拿到文本而缺少完整结构。
2. 目录总页数目前无法稳定从转换结果中精确恢复，因此目录页数检查仍以人工复核提示为主。
3. 测试项一致性、人员识别、POLQA 配置识别本质上仍是规则和关键词驱动，不等同于人工最终结论。

## 第二部分：当前模块结构

### 1. 主进程报告审查模块

当前报告审查主进程代码结构如下：

```text
src/main/services/reportReview/
├─ index.js
├─ wordReviewService.js
├─ utils.js
├─ reportBuilder.js
└─ checks/
   ├─ documentStructure.js
   ├─ metadata.js
   └─ polqa.js
```

职责划分：

- `index.js`
  - 作为报告审查模块统一入口
  - 调用 `parseReport()` 解析 Word 报告
  - 调用审查服务生成 `reviewResult` 和 `report`

- `wordReviewService.js`
  - 负责审查流程编排
  - 组织 8 项检查执行顺序
  - 汇总 `summary` 和 `overallStatus`

- `checks/documentStructure.js`
  - 目录提取
  - 目录页数检查
  - 章节与目录对应检查

- `checks/metadata.js`
  - 基本信息一致性
  - 测试项一致性
  - 名称污染检查
  - 人员识别

- `checks/polqa.js`
  - POLQA / MOS-LQO / P.863 相关配置检查

- `reportBuilder.js`
  - 将结构化检查结果转成前端展示使用的 section 列表

### 2. 前端报告审查页面模块

当前前端页面结构如下：

```text
src/renderer/pages/reportReviewModule/
├─ index.jsx
├─ ReviewResultContent.jsx
├─ reviewHistoryColumns.jsx
└─ constants.js
```

补充模块：

```text
src/renderer/modules/reportReview/
└─ storage.js
```

职责划分：

- `index.jsx`
  - 处理文件选择、拖拽、发起审查、历史读取、清空列表、详情弹窗

- `ReviewResultContent.jsx`
  - 渲染详细审查结果内容
  - 兼容历史记录中可能存在的不完整旧数据

- `reviewHistoryColumns.jsx`
  - 维护最近审查记录表格列定义

- `storage.js`
  - 管理报告审查历史的本地持久化读写
